Problem Statement & Context

DPDP Act Challenges
Core Problem

YouTube relies heavily on user data to drive recommendation accuracy. Under India's DPDP Act, 2023, legacy models of generic, bundled consent are no longer compliant. We must transition to explicit, granular, and revocable consent without degrading user engagement or causing conversion drop-offs.

Opportunities

Transforming a legal compliance mandate into user trust. By demonstrating high transparency and granular user control, YouTube can improve its user sentiment and set a new standard for Indian consumer tech.

Data Collected & Purposes
  • Watch & Search History: Powers personalized recommendation feeds.
  • Device Info & Location: Used for local language & regional content curation.
  • Advertising Interactions: Delivers higher advertiser relevance and ROI.
  • Voice Searches: Stores voice logs to improve accessibility search accuracy.
DPDP Notice Mandate: Consent request must specify what data is being collected and exactly what purpose it will serve.

DPDP Non-Compliance Audit
  • No Granular Control: User cannot consent to Watch History personalization while declining Location tracking. Consent is binary ("Take it or leave it").
  • Incomprehensible Terms: Privacy policies are long, complex text files, violating DPDP requirement for clear notice in plain language.
  • Buried Consent Revocation: DPDP specifies withdrawing consent must be as easy as giving it. Currently, revocation requires a 5-step external navigation.

Proposed Solution: PrivacyHub

Materializing DPDP Requirements

The Four Product Pillars

1. Purpose-Based Consent

Granular opt-ins for specific use cases (watch history, location, ads) with plain-language explanations.

2. PrivacyHub Dashboard

Self-service dashboard within Settings. Allows users to view status, download data, or delete accounts.

3. Child Protection

Verifiable parental consent for under-18 users. Block behavioral ads and location tracking for child accounts.

4. Just-in-Time Prompts

Contextual privacy cards explaining location and microphone needs at point of action (e.g. voice search clicks).
